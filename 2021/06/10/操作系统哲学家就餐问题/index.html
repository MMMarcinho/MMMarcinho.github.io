操作系统 - 哲学家就餐问题

哲学家吃饭问题

假设有 5 个哲学家，他们的生活只是思考和吃饭。这些哲学家共用一个圆桌，每位都有一把椅子。在桌子中央有一碗米饭，在桌子上放着 5 根筷子，如下图所示。

当一位哲学家思考时，他与其他同事不交流。时而，他会感到饥饿，并试图拿起与他相近的两根筷子（筷子在他和他的左或右邻居之间）。一个哲学家一次只能拿起一根筷子。显然，他不能从其他哲学家手里拿走筷子。当一个饥饿的哲学家同时拥有两根筷子时，他就能吃。在吃完后，他会放下两根筷子，并开始思考。

哲学家就餐问题是一个经典的同步问题，这不是因为其本身的实际重要性，也不是因为计算机科学家不喜欢哲学家，而是因为它是大量并发控制问题的一个例子。这个代表型的例子满足：在多个进程之间分配多个资源，而且不会出现死锁和饥饿。

一种简单的解决方法是每只筷子都用一个信号量来表示。一个哲学家通过执行操作 wait() 试图获取相应的筷子，他会通过执行操作 signal() 以释放相应的筷子。

首先来手动找到一个死锁

#include <pthread.h>
#include <stdio.h>
#include <unistd.h>
#include <stdlib.h>
#include <errno.h>
#include <assert.h>

#define PHILOS 5
#define DELAY 5000
#define FOOD 100

void *philosopher (void *id);
void grab_chopstick (int, int, char *);
void down_chopsticks (int, int);
int food_on_table ();

pthread_mutex_t chopstick[PHILOS];
pthread_t philo[PHILOS];
pthread_mutex_t food_lock;
pthread_mutex_t num_can_eat_lock;
int sleep_seconds = 0;
uint32_t num_can_eat = PHILOS - 1;


int main (int argn, char **argv)
{
    int i;
    pthread_mutex_init (&food_lock, NULL);
    pthread_mutex_init (&num_can_eat_lock, NULL);
    for (i = 0; i < PHILOS; i++)
        pthread_mutex_init (&chopstick[i], NULL);
    for (i = 0; i < PHILOS; i++)
        pthread_create (&philo[i], NULL, philosopher, (void *)i);
    for (i = 0; i < PHILOS; i++)
        pthread_join (philo[i], NULL);
    return 0;
}

void * philosopher (void *num)
{
    int id;
    int i, left_chopstick, right_chopstick, f;

    id = (int)num;
    printf ("Philosopher %d is done thinking and now ready to eat.\n", id);
    right_chopstick = id;
    left_chopstick = id + 1;

    /* Wrap around the chopsticks. */
    if (left_chopstick == PHILOS)
        left_chopstick = 0;

    while (f = food_on_table ()) {
        grab_chopstick (id, right_chopstick, "right ");
        grab_chopstick (id, left_chopstick, "left");
        printf ("Philosopher %d: eating.\n", id);
        usleep (DELAY * (FOOD - f + 1));
        down_chopsticks (left_chopstick, right_chopstick);
    }

    printf ("Philosopher %d is done eating.\n", id);
    return (NULL);
}

int food_on_table ()
{
    static int food = FOOD;
    int myfood;

    pthread_mutex_lock (&food_lock);
    if (food > 0) {
        food--;
    }
    myfood = food;
    pthread_mutex_unlock (&food_lock);
    return myfood;
}

void grab_chopstick (int phil, int c, char *hand)
{
    pthread_mutex_lock (&chopstick[c]);
    printf ("Philosopher %d: got %s chopstick %d\n", phil, hand, c);
}

void down_chopsticks (int c1, int c2)
{
    pthread_mutex_unlock (&chopstick[c1]);
    pthread_mutex_unlock (&chopstick[c2]);
}

如上代码什么保护都没做，可以说很容易就会产生五位哲学家同时抓起右手筷子的情况。

因为这里出现死锁太容易了，所以甚至可以把代码再完善一下，实现一个自动检测死锁的功能。

int chopsticks[5] = {0, 0, 0, 0, 0}; // 数组代表了每个哲学家右手边的筷子。0 代表了没有拿起，1 代表了拿起

void * philosopher (void *num)
{
    int id, i, left_chopstick, right_chopstick, f;
    id = (int)num;
    printf ("Philosopher %d is done thinking and now ready to eat.\n", id);
    right_chopstick = id;
    left_chopstick = id + 1;

    if (left_chopstick == PHILOS)
        left_chopstick = 0;

    while (f = food_on_table ()) {

        if (id == 1)
            sleep (sleep_seconds);
        grab_chopstick (id, right_chopstick, "right ");
        chopsticks[id] = 1; // 右手拿起筷子就设置为 1
        int sum = 0;
        for (int j = 0; j < 5; j ++)
        {
            sum += chopsticks[j];
            if (chopsticks[j] == 1) // 每次顺便检测哪些哲学家右手拿筷子
                printf("Philosopher %d: take his right chopstick  ", j);
        }
        printf("\n");
        if (sum == 5)
            printf("Dead Lock Now\n"); // 当五根右手筷子都被拿起时，就是出现死锁了
        grab_chopstick (id, left_chopstick, "left");
        printf ("Philosopher %d: eating.\n", id);
        usleep (DELAY * (FOOD - f + 1));
        down_chopsticks (left_chopstick, right_chopstick);
        printf ("Philosopher %d: puts down chopsticks.\n", id);
        chopsticks[id] = 0;
    }

    printf ("Philosopher %d is done eating.\n", id);
    return (NULL);
}

结果发现确实很快就自动检测到了死锁，并很快就进行不下去了，而且还显示出了各个哲学家拿筷子的顺序。

哲学家就餐问题防护死锁的方法

其实关于防护的话网上也有很多的解决方法，我们操作系统的老师亲切地教导我们原理并告诉了我们它们的名字。

顺序资源法

顺序资源法，在网上一搜会发现实际上大家都叫它有序资源法。其实他的原理很简单，也有几个不同的版本，但是核心思想都是一样的。例如我们上面的例子，我们可以让哲学家0 1 2 3号都右手先拿起筷子，而让哲学家4号做一个特立独行的人（怎么感觉也挺符合哲学家这个设定的），偏偏要让他左手先拿起筷子，这样在他想要拿起筷子时，先拿的就是哲学家3的右手的筷子，如果这跟筷子已经被拿走了，那他也不会去动自己右手的筷子，所以哲学家0号就可以很自如地拿起左右手的筷子，防护了死锁的问题。

void * philosopher (void *num)
{
    int id, i, left_chopstick, right_chopstick, f;
    id = (int)num;
    printf ("Philosopher %d is done thinking and now ready to eat.\n", id);
    if (id != 4)
    {
        right_chopstick = id;
        left_chopstick = id + 1;

        while (f = food_on_table ()) {
            grab_chopstick (id, right_chopstick, "right ");
            grab_chopstick (id, left_chopstick, "left");
            printf ("Philosopher %d: eating.\n", id);
            usleep (DELAY * (FOOD - f + 1));
            down_chopsticks (left_chopstick, right_chopstick);
        }

        printf ("Philosopher %d is done eating.\n", id);
        return (NULL);
    }
    else
    {
        right_chopstick = 4;
        left_chopstick = 0;

        while (f = food_on_table ()) {
            grab_chopstick (id, left_chopstick, "left");
            grab_chopstick (id, right_chopstick, "right ");
            printf ("Philosopher %d: eating.\n", id);
            usleep (DELAY * (FOOD - f + 1));
            down_chopsticks (right_chopstick, left_chopstick);
        }
        printf ("Philosopher %d is done eating.\n", id);
        return (NULL);
    }   
}

在这种情况下，从根本逻辑上就注定了不可能产生死锁问题，所以是一个常用的防护死锁的方法，我们的电脑里也经常用到这个方法（当然我们的电脑里没有一堆哲学家在吃干饭！）

可以看到程序运行得很顺利，不存在死锁的情况。

加房间法

其实我觉得这个名字取得就有点抽象了，明明就是强制让一个人没得吃，怎么就算加房间了呢。这里的方法很简单，就是强制使得只有只有 N-1 个哲学家同时想要就餐。这样设置之后，始终会有一个哲学家左右手的筷子都不会被他提起，希望他不会饿死。

int get_token ();
void return_token ();
uint32_t num_can_eat = PHILOS - 1;

void * philosopher (void *num){
    int id, i, left_chopstick, right_chopstick, f;
    id = (int)num;
    printf ("Philosopher %d is done thinking and now ready to eat.\n", id);
    right_chopstick = id;
    left_chopstick = id + 1;

    /* Wrap around the chopsticks. */
    if (left_chopstick == PHILOS)
        left_chopstick = 0;
    while (f = food_on_table ()) {
        get_token (); // 验证是否还有吃饭的空位了，没有就只能一直想

        grab_chopstick (id, right_chopstick, "right ");
        grab_chopstick (id, left_chopstick, "left");

        printf ("Philosopher %d: eating.\n", id);
        usleep (DELAY * (FOOD - f + 1));
        down_chopsticks (left_chopstick, right_chopstick);

        return_token (); // 吃完就把吃饭的空位让出来吧
    }

    printf ("Philosopher %d is done eating.\n", id);
    return (NULL);
}

int get_token (){
    int successful = 0;
    while (!successful) {
        pthread_mutex_lock (&num_can_eat_lock);
        if (num_can_eat > 0) {
            num_can_eat--;
            successful = 1;
        }
        else 
            successful = 0;
        pthread_mutex_unlock (&num_can_eat_lock);
    }
}

void return_token (){
    pthread_mutex_lock (&num_can_eat_lock);
    num_can_eat++;
    pthread_mutex_unlock (&num_can_eat_lock);
}

这种情况下确实会防护死锁，但是这个方法如果未经优化似乎会导致有的哲学家长时间吃不到饭，所以要是真实际运用起来还是要优化一下才行。

程序运行得很流畅，就是发现哲学家0好久没饭吃了，好不容易吃上了别的哲学家又没饭吃了……

Psim 法

这个名字我就不是特别理解了，但是方法从本意上很好理解，如上代码使得一次获得所有请求的信号量，只要任一个信号量不能获得，就一个也得不到，调用的进程也就被阻塞了。

void grab_chopstick (int, int, int);

void grab_chopstick (int phil, int c, int d)
{
    // 给筷子上锁，代表筷子被拿起
    while (pthread_mutex_trylock (&chopstick[c]) != 0 || pthread_mutex_trylock (&chopstick[d]) != 0);
    pthread_mutex_unlock(&chopstick[c]);
    pthread_mutex_unlock(&chopstick[d]);
    printf ("Philosopher %d: got chopstick %d and chopstick %d\n", phil, c, d);
}

这种情况下其实大家吃得都很均匀，相比之前的加房间法确实很不错，和顺序资源法相比似乎速度更快更流畅，但是我才疏学浅，不知道怎么比较这些方法的效率，希望以后学得更深入了能来补充一下。
